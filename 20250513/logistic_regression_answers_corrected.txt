1-3. 두 파라미터 후보의 가능도 비교

로지스틱 회귀에서 가능도는 다음과 같이 계산됩니다:
L(w) = ∏(h_w(x_i)^y_i * (1-h_w(x_i))^(1-y_i))

로그 가능도(log likelihood)는 다음과 같이 계산됩니다:
log L(w) = ∑(y_i*log(h_w(x_i)) + (1-y_i)*log(1-h_w(x_i)))

여기서:
- h_w(x) = 1/(1 + e^(-w·x))는 로지스틱 함수
- y_i는 실제 클래스 (1 또는 0)
- x_i는 입력 벡터 (x_0=1 포함)

주어진 데이터 포인트:
양성 예제(y=1): (1,2), (2,3)
음성 예제(y=0): (3,1), (4,2)

파라미터 후보:
w_1 = (0, -1, 1)
w_2 = (-3, -1, 3)

계산 결과:
w_1 = (0, -1, 1)의 로그 가능도: -0.8804
w_2 = (-3, -1, 3)의 로그 가능도: -0.5069

따라서 w_2가 더 높은 로그 가능도를 가지므로, w_2 = (-3, -1, 3)가 더 나은 파라미터입니다.

1-4. 클래스 경계 도출

[수정] 클래스 경계가 (2,1.5)와 (3,2.5)를 지나도록 해야 합니다.

로지스틱 회귀에서 클래스 경계는 h_w(x) = 0.5가 되는 지점입니다.
h_w(x) = 0.5일 때, w·x = 0 입니다.

두 점 (2,1.5)와 (3,2.5)을 지나는 경계선을 위한 파라미터를 구하겠습니다.
로지스틱 회귀 모델의 경계식은 w₀ + w₁x₁ + w₂x₂ = 0 형태입니다.

두 점을 경계식에 대입하면:
1) w₀ + w₁·2 + w₂·1.5 = 0
2) w₀ + w₁·3 + w₂·2.5 = 0

이 두 방정식을 만족하는 w = [w₀, w₁, w₂]를 구하기 위해,
식 2에서 식 1을 빼면:
w₁(3-2) + w₂(2.5-1.5) = 0
w₁ + w₂ = 0

w₂ = 1로 정규화하면, w₁ = -1이 됩니다.

이제 w₀를 구하면:
w₀ + (-1)·2 + 1·1.5 = 0
w₀ = 2 - 1.5 = 0.5

따라서 새로운 파라미터는 w = [0.5, -1, 1]입니다.

이 파라미터로 경계식을 다시 작성하면:
0.5 + (-1)x₁ + 1·x₂ = 0
x₂ = x₁ - 0.5

이 경계선은 정확히 (2,1.5)와 (3,2.5)를 지나는 것을 확인할 수 있습니다:
- x₁=2일 때: x₂ = 2 - 0.5 = 1.5 ✓
- x₁=3일 때: x₂ = 3 - 0.5 = 2.5 ✓

그리고 경계점(2,1.5)와 (3,2.5)를 함께 보여줍니다.

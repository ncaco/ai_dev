# 용어 정의

| 용어         | 약어   | 영문명                  | 설명                                                         |
|--------------|--------|------------------------|--------------------------------------------------------------|
| 자연어       | -      | Natural Language       | 인간이 일상적으로 사용하는 언어(예: 한국어, 영어 등)           |
| 자연어처리   | NLP    | Natural Language Processing | 컴퓨터가 자연어를 이해·분석·생성하도록 하는 인공지능 분야      |
| 자연어이해   | NLU    | Natural Language Understanding | 컴퓨터가 자연어의 의미를 이해하고 해석하는 기술 및 연구 분야      |
| 자연어생성   | NLG    | Natural Language Generation    | 컴퓨터가 자연어로 된 텍스트나 음성을 생성하는 기술 및 연구 분야   |
| 형태소분석   | -      | Morphological Analysis         | 문장을 구성하는 가장 작은 의미 단위(형태소)로 분해하고 품사 등을 분석하는 과정 |
| 통사분석     | -      | Syntactic Analysis             | 문장의 구조(구, 절, 문장 등)와 구성 요소 간의 관계를 분석하는 과정           |
| 의미분석     | -      | Semantic Analysis              | 문장이나 단어의 의미를 해석하고, 문맥에 따라 의미를 파악하는 과정            |
| 화용분석     | -      | Pragmatic Analysis             | 발화의 맥락, 의도, 상황 등 실제 사용 환경에서의 의미를 분석하는 과정         |
| 형태소       | -      | Morpheme                        | 의미를 가지는 가장 작은 언어 단위로, 단어를 구성하는 기본 요소(예: '먹', '는다', '고')를 의미함           |
| 품사태깅     | -      | Part-of-Speech Tagging         | 문장 내 각 단어의 품사(명사, 동사, 형용사 등)를 자동으로 식별하고 부착하는 과정. 형태소 분석의 결과로 주로 사용되며, 구문 분석 등 후속 자연어처리 작업의 기초가 됨. |
| 품사중의성   | -      | Part-of-Speech Ambiguity       | 하나의 단어가 문맥에 따라 여러 품사(예: 명사, 동사 등)로 해석될 수 있는 현상. 예: '눈'이 명사(eye) 또는 동사(see)의 의미로 사용될 수 있음. |
| 어휘중의성   | -      | Lexical Ambiguity              | 동일한 단어가 여러 의미(동음이의어)를 가질 수 있는 현상. 예: '배'가 과일, 신체 부위, 배(boat) 등 다양한 의미로 사용될 수 있음. |
| 규칙기반접근법 | -    | Rule-based Approach             | 언어학적 규칙(문법, 사전 등)을 사람이 직접 정의하여 자연어를 처리하는 방법. 규칙의 명확성과 해석 가능성이 높으나, 복잡한 언어 현상에 대한 확장성이 제한적임. |
| 통계기반접근법 | -    | Statistical Approach            | 대규모 말뭉치(코퍼스)에서 통계적 패턴을 학습하여 자연어를 처리하는 방법. N-gram, HMM, 머신러닝 등 다양한 통계적 기법이 활용됨. 데이터 기반으로 유연성이 높으나, 데이터 품질과 양에 따라 성능이 좌우됨. |
| 복합적접근법 | -     | Hybrid Approach                 | 규칙기반과 통계기반 접근법을 결합하여 각 방법의 단점을 보완하는 자연어처리 방법. 예를 들어, 규칙으로 전처리 후 통계적 모델을 적용하거나, 통계적 결과에 규칙을 추가로 적용하는 방식 등이 있음. |
| 트리구조       | -      | Tree Structure                  | 문장이나 구문을 계층적(트리 형태)으로 표현하는 구조. 각 노드는 구, 절, 단어 등 언어 단위를 나타내며, 상위-하위 관계를 통해 문장의 구조적 의미를 파악할 수 있음. |
| 파싱           | -      | Parsing                         | 입력된 문장(텍스트)을 문법 규칙에 따라 구조적으로 분석하여 트리구조 등으로 변환하는 과정. 구문 분석이라고도 하며, 컴퓨터가 문장의 구조를 이해하는 데 필수적임. |
| 구구조문법     | -      | Phrase Structure Grammar         | 문장을 구(명사구, 동사구 등) 단위로 분해하여 계층적 구조(트리)로 분석하는 문법 이론. 대표적으로 생성문법(Chomsky)이 있음. |
| 의존문법       | -      | Dependency Grammar               | 단어 간의 직접적인 의존 관계(지배-피지배)를 중심으로 문장 구조를 분석하는 문법 이론. 트리의 각 노드는 단어이며, 간선은 의존 관계를 나타냄. |
| 파싱알고리즘   | -      | Parsing Algorithm                | 문장의 구조를 분석(파싱)하기 위해 사용하는 알고리즘. 예: CKY, Earley, Shift-Reduce 등 다양한 방식이 존재함. |
| 차트파싱       | -      | Chart Parsing                    | 중간 결과(차트)를 저장하여 중복 계산을 피하고 효율적으로 파싱하는 알고리즘. 대표적으로 CKY, Earley 파서 등이 있음. 동적 프로그래밍 기법을 활용함. |
| 전이기반파싱   | -      | Transition-based Parsing         | 스택, 버퍼 등 자료구조를 사용하여 문장을 왼쪽에서 오른쪽으로 읽으며, 일련의 전이(shift, reduce 등) 동작을 통해 구문 트리를 점진적으로 구축하는 파싱 방식. 속도가 빠르고, 최근 신경망 기반 파서에서 많이 사용됨. |
| 오류전파       | -      | Error Propagation                | 파싱 또는 신경망 모델 등에서 발생한 오류가 이후 단계로 전이되어 전체 결과에 영향을 미치는 현상. 예: 잘못된 품사 태깅이 구문 분석 오류로 이어질 수 있음. |
| 조합폭발       | -      | Combinatorial Explosion          | 가능한 경우의 수(조합)가 기하급수적으로 증가하여 계산량이 급격히 커지는 현상. 자연어 파싱에서 모든 가능한 구문 구조를 탐색할 때 발생할 수 있음. 효율적인 알고리즘 설계가 필요함. |
| 어의중의성     | -      | Semantic Ambiguity              | 하나의 문장이나 단어가 문맥에 따라 여러 가지 의미(해석)를 가질 수 있는 현상. 예: "은행"이 금융기관(bank) 또는 강둑(river bank)을 의미할 수 있음. |
| 어의중의성해소 | WSD    | Word Sense Disambiguation       | 문맥 정보를 활용하여 다의어(여러 의미를 가진 단어)의 올바른 의미를 자동으로 판별하는 자연어처리 기술. |
| 지식기반중의성해소 | -   | Knowledge-based WSD             | 사전, 시소러스, 온톨로지 등 외부 지식 자원을 활용하여 어의중의성을 해소하는 방법. 예: WordNet 기반 알고리즘 등. |
| 감독중의성해소 | Supervised WSD | Supervised Word Sense Disambiguation | 의미가 주석된 학습 데이터를 이용해 기계학습 모델을 훈련하여 어의중의성을 해소하는 방법. 높은 정확도를 보이나, 대규모 주석 데이터가 필요함. |
| 비감독중의성해소 | Unsupervised WSD | Unsupervised Word Sense Disambiguation | 의미 주석 없이 말뭉치 내 통계적 패턴이나 클러스터링을 활용해 어의중의성을 해소하는 방법. 데이터 구축 비용이 적으나, 성능이 제한적일 수 있음. |
| 기계번역       | MT     | Machine Translation              | 한 언어로 된 텍스트를 다른 언어로 자동 변환하는 기술. 통계적, 규칙기반, 신경망 기반 등 다양한 방식이 존재하며, 대표적으로 구글 번역, 파파고 등이 있음. |
| 정보검색       | IR     | Information Retrieval            | 대용량 데이터(문서, 웹 등)에서 사용자의 질의에 맞는 관련 정보를 효과적으로 찾아내는 기술. 검색 엔진, 문서 검색 시스템 등에 활용됨. |
| 질의응답       | QA     | Question Answering               | 사용자의 질문에 대해 관련 정보를 분석·이해하여 정확한 답변을 자동으로 생성하거나 추출하는 기술. 챗봇, 검색 기반 QA, 생성형 QA 등으로 구분됨. |
| 음성인식       | ASR    | Automatic Speech Recognition      | 사람의 음성 신호를 텍스트로 변환하는 기술. 음성 비서, 자동 자막 생성, 콜센터 등에서 활용됨. |
| 음성합성       | TTS    | Text-to-Speech                   | 텍스트 데이터를 자연스러운 음성으로 변환하는 기술. 내비게이션, 음성 안내, AI 스피커 등에 사용됨. |
| 문서축약       | -      | Text Summarization                | 긴 문서나 텍스트에서 핵심 내용을 자동으로 요약하는 기술. 추출적 요약과 생성적 요약 방식이 있음. |
| 맞춤법검사     | -      | Spell Checking                    | 텍스트 내의 맞춤법 오류를 자동으로 탐지하고 수정하는 기술. 워드 프로세서, 웹 에디터 등에 적용됨. |
| 문법검사       | -      | Grammar Checking                  | 문장 내의 문법적 오류(어순, 시제, 조화 등)를 자동으로 탐지하고 교정하는 기술. 맞춤법 검사와 함께 사용되어 문서 품질을 높임. |
